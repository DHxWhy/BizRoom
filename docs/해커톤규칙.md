# AI Dev Days Hackathon에 오신 것을 환영합니다

AI Dev Days Hackathon은 AI Dev Days의 후속 이벤트입니다. 세션, 데모, 심층 분석을 즐기셨다면, 이제 영감을 실제 혁신으로 전환할 차례입니다. AI Dev Days에서 배운 것을 Microsoft Foundry, Microsoft Agent Framework, Azure MCP, GitHub Copilot Agent Mode 등을 활용해 프로덕션 수준의 솔루션으로 만들어 보세요.

이 해커톤은 **5주간 진행되는 글로벌 챌린지**로, 개발자가 Microsoft의 AI 플랫폼, GitHub, VS Code, Azure 서비스를 활용하여 AI 에이전트, Agentic DevOps 워크플로우, 지능형 애플리케이션을 구축하는 대회입니다.

---

## 목적에 맞게 설계된 AI 플랫폼으로 구축하세요

Microsoft의 AI 플랫폼은 실제 프로덕션 수준의 AI 시스템 구축을 위한 엔드투엔드 도구 체인을 개발자에게 제공합니다. Microsoft의 최신 AI, 에이전트, 클라우드, 개발자 기술을 활용하여 모든 산업 분야의 실제 문제를 해결하는 솔루션을 구축하세요.

다음과 같은 것을 만들 수 있습니다:

- Microsoft Agent Framework를 사용한 **AI 에이전트**
- GitHub Copilot Agent Mode를 활용한 **Agentic DevOps 워크플로우**
- Model Router 기반의 **멀티 에이전트 애플리케이션**
- Microsoft Foundry를 사용한 **보안 엔터프라이즈 AI 앱**
- Azure MCP를 활용한 **Agent-to-Agent(A2A) 또는 MCP 연결 시스템**
- Azure SRE Agent를 사용한 **차세대 SRE 워크플로우**

이 해커톤이 여러분의 실험장입니다.

---

## 무엇을 만들어야 하는가

### 핵심 요구사항

1. **Hero 기술 사용**: Microsoft Foundry, Microsoft Agent Framework, Azure MCP, GitHub Copilot Agent Mode 중 하나 이상을 활용한 AI 기반 솔루션을 구축할 것
2. **Azure 배포**: 애플리케이션은 Azure 서비스를 활용하고, Azure 클라우드 플랫폼에 배포 가능해야 함
3. **GitHub 활용 개발**: 프로젝트는 공개(public) GitHub 리포지토리에 호스팅되어야 하며, VS Code 또는 Visual Studio로 개발하고, GitHub Copilot으로 개선할 것

## 제출 항목

제출물에는 다음이 포함되어야 합니다:

1. **작동하는 프로젝트** — 필수 개발 도구로 구축되고 핵심 요구사항을 충족하는 프로젝트
2. **프로젝트 설명** — 기능, 해결한 문제, 사용된 기술을 설명하는 문서
3. **데모 영상** (최대 2분) — 프로젝트 동작을 보여주는 영상. YouTube 또는 Vimeo에 업로드 후 공개 링크 제공. 허가 없이 제3자의 상표 또는 저작권 자료를 포함해서는 안 됨
4. **공개 GitHub 리포지토리** — 심사 및 테스트를 위한 소스 코드
5. **아키텍처 다이어그램** — 솔루션이 Microsoft Foundry, Agent Framework, Azure MCP, GitHub Copilot, Azure 서비스를 어떻게 활용하는지 보여주는 구조도
6. **팀원 정보** — 모든 참가자의 Microsoft Learn 사용자명 포함

## 심사 기준

프로젝트는 **5개 동일 가중치 항목**(각 20%)으로 평가됩니다:

### 1. 기술 구현 (20%)

- 높은 수준의 소프트웨어 개발 관행
- Hero 기술의 효과적 활용
- 구조화되고, 문서화되고, 유지보수 가능한 코드

### 2. Agentic 설계 및 혁신 (20%)

- Agentic AI 패턴의 창의적 적용
- 정교한 에이전트 오케스트레이션 또는 멀티 에이전트 협업
- 새로운 AI 구현 또는 의미 있는 개선

### 3. 실제 영향력 및 적용 가능성 (20%)

- 해결하고자 하는 문제의 중요성
- 프로덕션 배포 준비 상태
- 개발자, 기업, 최종 사용자에 대한 잠재적 영향력

### 4. 사용자 경험 및 프레젠테이션 (20%)

- 잘 설계되고 직관적인 사용자 경험
- 데모에서 프로젝트 가치의 명확한 전달
- 프론트엔드와 백엔드의 균형 잡힌 구현

### 5. 해커톤 카테고리 준수 (20%)

- 해당 해커톤 카테고리 설명에 부합할 것
