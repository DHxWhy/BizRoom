## 1. Executive Summary

### 1.1 제품 비전

**BizRoom.ai**는 1인 사업자부터 소규모 팀까지, AI C-Suite 임원진과 실시간 그룹 회의를 할 수 있는 가상 사무실이다.

> "직원 1명인 회사에 CEO 빼고 전원 AI 임원. CFO가 재무제표 만들고, CMO가 마케팅 전략 짜고, CTO가 코드 리뷰한다."

### 1.2 핵심 가치

| 가치               | 설명                                                                 |
| ------------------ | -------------------------------------------------------------------- |
| 전문성 민주화      | 경영·재무·마케팅·법무·기술 전문가를 누구나 즉시 활용                  |
| N:N 집단 지성      | 1:1 챗봇이 아닌, 다수 에이전트 간 토론·반박·보완을 통한 의사결정     |
| 실시간 산출물      | 회의 중 재무제표, 마케팅 플랜, 회의록 등이 즉시 생성                 |
| 인간 중심 의사결정 | AI는 분석·제안하고, 최종 결정은 항상 사용자(Chairman)가 내린다       |

### 1.3 해커톤 컨텍스트

| 항목             | 내용                                                           |
| ---------------- | -------------------------------------------------------------- |
| 대회             | Microsoft AI Dev Days Hackathon                                |
| 마감             | 2026년 3월 15일 (PST)                                          |
| 상금 풀          | $80K+                                                          |
| 타겟 카테고리    | Grand Prize ($20K) + Best Multi-Agent ($10K) + Enterprise ($10K) |
| 핵심 기술        | Semantic Kernel + Azure AI Foundry + A2A Protocol              |

## 2. Problem Statement

### 2.1 문제 정의

1인 사업자와 소규모 팀은 경영, 재무, 마케팅, 법무, 기술 등 다방면의 전문 영역을 소수 인원이 동시에 감당해야 한다. 이로 인해 **전문성 부족**, **의사결정 품질 저하**, **번아웃**이 구조적으로 발생한다.

### 2.2 시장 데이터

| 지표                               | 수치                     | 출처                           |
| ---------------------------------- | ------------------------ | ------------------------------ |
| 미국 1인 사업자 수                 | 5,300만 명               | Bureau of Labor Statistics     |
| 글로벌 프리랜서 시장 규모          | $1.5T (2025)             | Statista                       |
| 소규모 비즈니스 실패율 (5년 내)    | 50%                      | SBA                            |
| 실패 사유 중 "전문성 부족" 비율    | 42%                      | CB Insights                    |
| 글로벌 AI 에이전트 시장 CAGR      | 44.8% (2024-2030)        | Grand View Research            |
| 전문 컨설팅 시간당 비용 (미국 평균) | $150-500/hr              | Consultancy.org                |

### 2.3 기존 솔루션의 한계

| 기존 솔루션          | 한계                                                              |
| -------------------- | ----------------------------------------------------------------- |
| ChatGPT / Copilot    | 1:1 명령→응답 패턴. 다각적 검토, 팀 시뮬레이션 불가               |
| Notion AI / Jasper   | 문서 보조 도구. 의사결정 토론 기능 없음                           |
| 경영 컨설팅          | 시간당 $150-500. 1인 사업자가 감당 불가                           |
| 멀티에이전트 프레임워크 | 개발자 도구 수준. 비기술 사용자가 직접 사용 불가                |

### 2.4 핵심 갭 (Gap)

```
[사용자의 현실]                         [사용자가 필요한 것]
1인이 모든 결정을 혼자 내림      →      다각적 전문가 검토 후 의사결정
AI에게 1:1로 질문                →      전문가 팀이 서로 토론하며 최적안 도출
결과물을 직접 만듦               →      회의 중 산출물이 자동 생성
혼자 고립된 의사결정             →      팀 역학이 있는 협업 환경
```

## 3. Solution Overview

### 3.1 제품 개요

BizRoom.ai는 Google Research의 **DialogLab** 개념을 적용한 멀티에이전트 그룹 대화 환경이다. Slack 스타일의 채팅 인터페이스에서 AI C-Suite 임원진(COO, CFO, CMO, CTO, CDO, CLO)이 실시간으로 토론하며, 사용자(Chairman)가 최종 의사결정을 내린다.

### 3.2 핵심 차별점

| #   | 차별점                         | 설명                                                         |
| --- | ------------------------------ | ------------------------------------------------------------ |
| 1   | N:N 대화                      | 1:1 챗봇이 아닌, 다수 에이전트 + 다수 인간의 그룹 회의       |
| 2   | 에이전트 간 자율 토론          | 에이전트들이 서로 반박·보완하며 논점을 다각적으로 검토        |
| 3   | 실시간 산출물 생성             | 회의 중 Excel, 회의록, 마케팅 플랜 등이 즉시 생성            |
| 4   | DialogLab 기반 턴테이킹        | 사회적 역학(turn-taking, social dynamics)이 적용된 대화 구조  |
| 5   | 혼합 회의                      | 실제 팀원 + AI 에이전트가 동일 회의에서 협업                  |

### 3.3 아키텍처 개요

```
┌──────────────────────────────────────────────────────────────┐
│                        Frontend                              │
│    React + Tailwind (Slack-style Chat UI)                    │
│    Web Speech API (Push-to-Talk STT)                        │
│    Azure Static Web Apps (Deploy)                           │
├──────────────────────────────────────────────────────────────┤
│                      Realtime Layer                          │
│    Azure SignalR Service (WebSocket)                        │
│    GPT Realtime API (WebRTC, v2)                            │
├──────────────────────────────────────────────────────────────┤
│                        Backend                               │
│    Azure Functions (Node.js)                                │
│    Semantic Kernel (Microsoft Agent Framework)              │
│    A2A Protocol (Agent-to-Agent)                            │
│    MCP (Model Context Protocol)                             │
│    Microsoft Graph API (OneDrive, Calendar, Planner)        │
├──────────────────────────────────────────────────────────────┤
│                       AI Layer                               │
│    Azure AI Foundry Model Router                            │
│    GPT-4o / GPT-4o-mini / DALL-E 3 / GPT Realtime          │
│    Bing Search API (사전 조사 + 실시간 그라운딩)             │
└──────────────────────────────────────────────────────────────┘
```

## 5. Core Features

### 5.1 Feature 1: Meeting Mode (그룹 회의)

**개요**: Slack 스타일 그룹 채팅 인터페이스에서 AI 임원진과 실시간 회의를 진행한다.

| 속성             | 내용                                                              |
| ---------------- | ----------------------------------------------------------------- |
| 우선순위         | P0 (MVP 필수)                                                     |
| 인터랙션 패턴    | 사용자가 안건 제시 → 에이전트들이 순서대로 의견 → 토론 → 결정     |
| UI 레퍼런스      | Slack 채널 뷰 (타임라인 + 아바타 + 이름 + 메시지)                 |
| 턴테이킹 방식    | DialogLab 기반: 관련 도메인 에이전트 우선 → 보충 의견 → 반박      |

**상세 동작 흐름**:

0. (Phase 0) COO가 Bing Search API로 안건 관련 사전 조사를 수행하고 결과를 공유한다.
1. 사용자가 조사 결과를 확인하고 회의 시작을 승인한다.
2. COO가 안건을 분석하고, 관련 에이전트를 지정한다.
3. 지정된 에이전트들이 순서대로 의견을 제시한다.
4. 에이전트 간 반박/보완이 발생한다 (자율 토론).
5. 사용자가 퀵 액션(동의/반대/다음/보류)으로 방향을 제어한다.
6. 결론 도출 시 COO가 요약하고 액션 아이템을 정리한다.

**에이전트 응답 규칙**:

| 규칙             | 설명                                                              |
| ---------------- | ----------------------------------------------------------------- |
| 응답 길이        | 1회 발언 최대 200 토큰. 간결하고 핵심 위주                        |
| 턴 순서          | 도메인 관련도 높은 에이전트 우선. COO가 조율                      |
| 자율 반박        | 다른 에이전트 의견에 동의하지 않으면 자동 반박 발생                |
| 산출물 트리거    | "만들어줘", "생성해줘" 등 키워드 시 Artifact 자동 생성             |
| 멘션 시스템      | @CFO, @CTO 등으로 특정 에이전트 지정 가능                         |

### 5.2 Feature 2: Push-to-Talk 음성 입력

**개요**: 스페이스바를 길게 누르면 음성 입력이 시작되고, 떼면 STT 변환되어 텍스트로 전송된다. MVP에서는 Web Speech API를 사용하며, Level 2/3에서는 GPT Realtime API로 전환하여 에이전트가 음성으로 직접 응답하는 풀 voice-in/voice-out 대화를 지원한다.

> **v2 계획**: GPT Realtime API (WebRTC)로 업그레이드하여 에이전트가 음성으로 직접 응답하고, 사용자가 끊어 말하면 자동 인터럽트되는 실제 대화 경험을 구현한다.

| 속성             | 내용                                                              |
| ---------------- | ----------------------------------------------------------------- |
| 우선순위         | P0 (MVP 필수)                                                     |
| 입력 방식        | Space bar hold → 녹음 → release → STT → 텍스트 전송              |
| STT 엔진         | Web Speech API (브라우저 네이티브)                                 |
| 지원 언어        | 한국어, 영어 (MVP). 추후 다국어 확장                              |
| 화자 식별        | 멀티유저 환경에서 발화자 자동 식별 (세션 기반)                     |

**UX 마찰 분석**:

| 단계                              | 마찰 수준 | 대응                                     |
| --------------------------------- | --------- | ---------------------------------------- |
| Space 홀드                        | 낮음      | 워키토키 UX. 직관적                      |
| STT 변환 대기                     | 중간      | 실시간 텍스트 프리뷰 표시                |
| 오인식 수정                       | 높음      | 전송 전 수정 기회 제공 (2초 딜레이)      |
| 텍스트 폴백                       | 없음      | 언제든 텍스트 입력 가능                  |

### 5.3 Feature 3: Multi-user 회의

| 속성                 | 값                                                              |

```
    │
    └─ [에이전트들이 자율적으로 30분간 토론]
        ├─ CMO: 시장 분석, 경쟁사 조사, 포지셔닝 전략
        ├─ CFO: 진출 비용 추정, 환율 리스크, 자금 조달
        ├─ CTO: 인프라 현지화, 규제 기술 요건
        ├─ CLO: 미국 사업자 등록, 세법, 개인정보보호법
        ├─ CDO: 현지화 UI/UX, 브랜드 어댑테이션
        └─ COO: 타임라인, 마일스톤, 리소스 배분
    │
    └─ [COO] 최종 보고서 + 액션 플랜 자동 생성 → 사용자에게 전달
```

## 부록

### A. DialogLab 적용 원칙

Google Research의 DialogLab에서 차용하는 핵심 원칙:

1. **턴테이킹 (Turn-Taking)**: 에이전트 간 자연스러운 발언 순서 관리
2. **사회적 역학 (Social Dynamics)**: 동의, 반박, 보완 등 사회적 상호작용 패턴
3. **역할 기반 발언 (Role-Based Contribution)**: 전문 분야에 따른 발언 트리거
4. **갈등 해결 (Conflict Resolution)**: 에이전트 간 의견 충돌 시 구조적 토론 진행

> **출처**: Google Research, "Beyond One-on-One: Authoring, Simulating, and Testing Dynamic Human-AI Group Conversations" (ACM UIST 2025). BizRoom.ai는 이 연구의 개념을 Microsoft 생태계 위에서 재구현한다.
