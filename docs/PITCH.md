# BizRoom.ai — 프로젝트 설명 (Hackathon Submission)

> **"직원 1명인 회사에 CEO 빼고 전원 AI 임원."**

---

## 1. 한 줄 소개

**"What if every solo founder had a full C-Suite — inspired by the minds behind Microsoft?"**

모든 1인 창업자에게 Microsoft 리더십에서 영감을 받은 AI C-Suite 임원진을 제공하는 가상 회의실, **BizRoom.ai**.

---

## 2. 문제 (Problem)

### 혼자 모든 걸 해야 하는 사람들

전 세계적으로 프리랜서와 1인 사업자가 폭발적으로 증가하고 있습니다.

| 지표                              | 수치                         | 출처               |
| --------------------------------- | ---------------------------- | ------------------ |
| 미국 프리랜서 수                  | 5,300만 명+ (2024)           | Upwork             |
| 미국 내 1인 사업자(Solopreneur)   | 2,800만+ 개 사업체           | US Census Bureau   |
| 글로벌 기그 이코노미 규모         | $5,560억 (2025 예상)         | Statista           |
| 1인 사업자 평균 업무 시간         | 주 52시간                    | FreshBooks         |

### 이들이 겪는 고통

- **경영, 재무, 마케팅, 법무, 기술을 혼자 감당** — 전문성 부족으로 의사결정 품질 저하
- **의사결정의 고독** — 중요한 결정을 내릴 때 논의할 팀이 없다. 혼자 결정하면 blind spot이 생긴다
- **번아웃** — 모든 역할을 혼자 수행하며 신체적·정신적 소진
- **비용의 벽** — 실제 임원을 고용하면 인당 연 $150K+, 컨설팅은 시간당 $300+

### 기존 AI 도구의 한계

- **1:1 챗봇 패러다임** — ChatGPT, Copilot 등은 단일 AI와 1:1 대화
- **다각적 검토 불가** — 하나의 AI가 재무·마케팅·운영 관점을 동시에 균형 있게 다루기 어려움
- **팀 시뮬레이션 불가** — 서로 반박하고 보완하는 회의 경험을 제공하지 못함
- **"질문하는 도구"에 머물러 있다** — 사용자가 프롬프트를 잘 써야 좋은 답을 얻음

> "창업은 팀 스포츠인데, 나는 혼자 뛰고 있다."

## 4. 작동 방식 (How It Works)

### 4-1. Meeting Mode — 그룹 채팅에서 AI 임원들이 토론

사용자(Chairman)가 안건을 제시하면, AI 임원진이 자신의 전문 영역에서 의견을 제시합니다. 에이전트들은 서로의 의견을 참조하고, **반박**하고, **보완**합니다.

```
Chairman → 안건 제시: "이번 주 계획 세워줘"
  ↓
COO Hudson → 안건 3개 정리, 실행 우선순위 제안
  ↓
CFO Amelia → 이번 달 예산 잔여 현황 보고, 비용 리스크 지적
  ↓
CMO Yusef → Product Hunt 런칭 타이밍 제안, GTM 전략
  ↓
CFO Amelia → CMO Yusef의 제안에 예산 관점 반박/보완
  ↓
COO Hudson → 양쪽 의견 종합, 실행 계획 확정
  ↓
Chairman → 최종 결정 / 추가 질문 / 아티팩트 요청
```

### 4-2. Push-to-Talk — 스페이스바를 누르고 말하면 된다

Web Speech API를 활용한 음성 입력. 스페이스바를 누르고 말하면 텍스트로 변환되어 회의에 전송됩니다. 키보드 없이도 진짜 회의처럼 대화할 수 있습니다.

### 4-3. Real-time Artifacts — 회의 중 실무 문서 즉시 생성

| 에이전트       | 아티팩트               | 형식       |
| -------------- | ---------------------- | ---------- |
| COO Hudson     | 회의록, 액션 아이템    | Markdown   |
| CFO Amelia     | 재무 분석, 예산표      | Excel      |

회의가 끝나면 산출물이 이미 준비되어 있습니다.

### 4-4. Multi-user — 실제 팀원 + AI 임원 혼합 회의

Azure SignalR을 통해 여러 사용자가 동시에 같은 회의에 참여할 수 있습니다. 실제 팀원과 AI 임원이 한 회의실에서 협업합니다.

## 5. 혁신 (Innovation)

### 핵심 혁신 포인트

| 혁신 요소                              | 설명                                                                       |
| -------------------------------------- | -------------------------------------------------------------------------- |
| **DialogLab → 프로덕트 전환**          | Google Research 학술 연구를 실제 비즈니스 프로덕트로 최초 적용              |
| **N:N 멀티파티 대화**                  | 1:1 챗봇을 넘어선 다자간 토론 시스템. 에이전트 간 반박/보완/협업           |
| **Turn-Taking 엔진**                   | 에이전트가 적절한 시점에 발언. 모든 에이전트가 무조건 답하지 않음           |
| **Interruption & Backchanneling**      | 긴급한 이슈에 끼어들기, 동의/이해 표시 등 자연스러운 대화 패턴             |
| **MS 리더십 가치관 기반 페르소나**     | 에이전트가 일관된 의사결정 프레임워크를 가짐 — 단순 역할 연기가 아닌 철학   |
| **진짜 회의 시뮬레이션**               | 명령→응답이 아닌, 토론→합의→산출물 흐름                                    |

### 기존 AI 챗봇과의 근본적 차이

```
기존 AI 챗봇:               BizRoom.ai:

사용자 → AI                 Chairman → COO Hudson ↔ CFO Amelia ↔ CMO Yusef
  ↑      ↓                       ↑              ↕
  └──────┘                       └───────── 다자간 토론

1:1 대화                    1:N 회의 (Multi-Party)
단일 관점                   다중 전문 관점 (반박/보완)
수동적 응답                 능동적 토론
프롬프트 작성자             Chairman (의사결정자)
```

| **GitHub Copilot**                       | 개발 전 과정에서 Copilot Agent Mode 활용. 에이전트 코드, UI 컴포넌트, 테스트 작성          |
