## 1. 디자인 철학

### 1.1 핵심 원칙

BizRoom.ai의 디자인은 세 가지 축으로 구성된다.

| 축                       | 원칙                                                                        |
| ------------------------ | --------------------------------------------------------------------------- |
| Microsoft Fluent 2       | Windows 11 / Microsoft 365의 최신 디자인 언어를 충실히 반영                 |
| AI-Native UX             | Microsoft Copilot에서 검증된 AI 상호작용 패턴 차용                         |
| 그룹 채팅 UX             | Teams / Slack에서 입증된 실시간 협업 패턴을 기반으로 확장                   |

### 1.2 Fluent Design System 2 (2024-2026)

Microsoft의 공식 디자인 시스템인 Fluent 2의 5가지 핵심 요소를 BizRoom에 적용한다.

| Fluent 2 요소            | BizRoom 적용                                                                |
| ------------------------ | --------------------------------------------------------------------------- |
| **Depth (깊이)**         | 사이드바, 메시지 카드, 아티팩트에 레이어와 그림자로 공간감 부여             |
| **Motion (모션)**        | 메시지 진입, 타이핑 인디케이터, 페이즈 전환에 자연스러운 애니메이션         |
| **Material (재질)**      | 사이드바에 Mica 효과, 모달에 Acrylic 반투명 효과 적용                      |
| **Light (빛)**           | 에이전트 컬러 그라데이션, 활성 상태 하이라이트                              |
| **Scale (스케일)**       | Desktop → Tablet → Mobile 반응형 레이아웃                                  |

### 1.3 Windows 11 / Microsoft 365 디자인 언어

| 특성                     | 상세                                                                        |
| ------------------------ | --------------------------------------------------------------------------- |
| 둥근 모서리              | 기본 8px, 메시지 버블 12px, 필(pill) 버튼 20px                             |
| 부드러운 그림자          | 3단계 elevation (subtle → medium → strong)                                 |
| 넉넉한 여백              | 기본 간격 단위 4px, 섹션 간 16~24px 여백                                   |
| 절제된 색상              | 모노크롬 기반 + 브랜드 컬러 포인트, 과도한 색상 사용 지양                  |
| 높은 대비                | 텍스트-배경 간 최소 4.5:1 대비율 유지                                      |

### 1.4 Microsoft Copilot UI 트렌드 (2025-2026)

AI 응답을 시각적으로 차별화하는 최신 Copilot 디자인 패턴을 적극 차용한다.

| 패턴                     | BizRoom 구현                                                                |
| ------------------------ | --------------------------------------------------------------------------- |
| AI 응답 시각적 구분      | 에이전트별 좌측 컬러 보더(3px) + 미세한 배경색 틴트                        |
| 스트리밍 텍스트           | 한 글자씩 타이핑되는 애니메이션 (SSE 기반 실시간 렌더링)                   |
| 카드형 산출물             | 아티팩트(Excel, PDF 등)를 인라인 카드로 표시, 프리뷰 + 다운로드            |
| "생각 중..." 인디케이터  | 에이전트 아바타 + bouncing dots + "Hudson가 정리하고 있습니다..." 텍스트     |
| 인라인 참조              | 회의 안건, 이전 발언 등을 인라인 링크로 참조 가능                          |

| CFO    | Amelia     | `#107C10` | 재무 / 안정의 그린         | 💰       |
| CMO    | Yusef     | `#FF8C00` | 마케팅 / 에너지의 오렌지   | 📣       |
| CTO    | Kelvin     | `#5C2D91` | 기술 / 혁신의 퍼플         | 💻       |

## 5. 컴포넌트 디자인 상세

### 5.1 Message Bubble (메시지 버블)

채팅의 핵심 컴포넌트. 발신자 유형에 따라 3가지 변형이 있다.

#### 에이전트 메시지

```
┌─────────────────────────────────────────────────┐
│ ┃ [Avatar]  에이전트명 (역할)     타임스탬프     │
│ ┃                                                │
│ ┃  메시지 내용이 여기에 표시됩니다.              │
│ ┃  여러 줄로 이어질 수 있습니다.                │
│ ┃                                                │
│ ┃  [Artifact Card - optional]                    │
└─────────────────────────────────────────────────┘
```

| 속성                     | 값                                                       |
| ------------------------ | -------------------------------------------------------- |
| 정렬                     | 좌측 정렬                                                |
| 좌측 보더                | 3px solid {에이전트 컬러}                                |
| 배경                     | `#FFFFFF` (light) / `#2D2D2D` (dark) + 에이전트 틴트    |
| 아바타                   | 40x40px, 원형, 에이전트 컬러 배경 + 아이콘               |
| border-radius            | 12px (좌상단 2px — 보더와 만나는 쪽)                     |
| 패딩                     | 12px 16px                                                |
| 최대 너비                | 72% of chat area                                         |
| 호버 시                  | 타임스탬프 표시, 반응 버튼 팝업                          |

#### 사용자(Human) 메시지

```
┌─────────────────────────────────────────────────┐
│     타임스탬프  🎙️  사용자명       [Avatar]      │
│                                                  │
│     메시지 내용이 여기에 표시됩니다.             │
│     여러 줄로 이어질 수 있습니다.               │
└─────────────────────────────────────────────────┘
```

| 속성                     | 값                                                       |
| ------------------------ | -------------------------------------------------------- |
| 정렬                     | 우측 정렬                                                |
| 배경                     | `#0078D4` (Brand Blue)                                   |
| 텍스트 색상              | `#FFFFFF`                                                |
| 아바타                   | 40x40px, 원형, 이니셜 레터                               |
| border-radius            | 12px (우상단 2px)                                        |
| 패딩                     | 12px 16px                                                |
| 최대 너비                | 72% of chat area                                         |
| 음성 입력 표시           | 🎙️ 아이콘 (이름 옆)                                     |

#### 다른 참여자 메시지

| 속성                     | 값                                                       |
| ------------------------ | -------------------------------------------------------- |
| 정렬                     | 좌측 정렬                                                |

## 6. 모션 & 애니메이션

### 6.1 Easing Functions

| 이름                     | 값                              | 용도                             |
| ------------------------ | ------------------------------- | -------------------------------- |
| ease-standard            | `cubic-bezier(0.4, 0, 0.2, 1)` | 일반 전환 (대부분의 애니메이션)  |
| ease-accelerate          | `cubic-bezier(0.4, 0, 1, 1)`   | 퇴장 애니메이션 (사라지는 요소)  |
| ease-decelerate          | `cubic-bezier(0.0, 0, 0.2, 1)` | 등장 애니메이션 (나타나는 요소)  |
| ease-bounce              | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 강조 / 바운스 효과          |

### 6.2 애니메이션 상세

| 애니메이션                       | duration | easing          | 상세                                              |
| -------------------------------- | -------- | --------------- | ------------------------------------------------- |
| 메시지 진입                      | 200ms    | ease-decelerate | `translateY(12px)` → `translateY(0)` + opacity     |
| 타이핑 인디케이터 점             | 1.4s     | ease-standard   | 3점 순차 bounce, 각 200ms delay                   |
| 에이전트 응답 스트리밍           | per char | linear          | 텍스트가 한 글자씩 나타남 (SSE stream 기반)        |
| 사이드바 토글                    | 250ms    | ease-standard   | `translateX(-260px)` → `translateX(0)`             |
| Push-to-Talk 펄스               | 1.5s     | ease-standard   | `scale(1)` → `scale(1.2)` + opacity 펄스, 반복    |
| Artifact 카드 등장               | 300ms    | ease-bounce     | `translateY(-8px)` + `scale(0.95)` → 정상 위치     |
| Meeting Phase 전환               | 500ms    | ease-standard   | 배경 그라데이션 컬러 변경                          |
| 호버 elevation 상승              | 150ms    | ease-standard   | box-shadow 변경                                    |
| Focus ring                       | 100ms    | ease-standard   | 2px Brand Blue outline 등장                        |
| Toast 알림 진입                  | 300ms    | ease-decelerate | 우측에서 슬라이드 인                               |
| Toast 알림 퇴장                  | 200ms    | ease-accelerate | 우측으로 슬라이드 아웃                             |

```css
@media (prefers-reduced-motion: reduce) {
```

| elevation-2 | `0 2px 4px rgba(0,0,0,0.14)`       | `0 2px 4px rgba(0,0,0,0.5)` + 상단 1px 밝은 선 |

### 11.3 다크 모드 토글 로직

```typescript
// src/hooks/useTheme.ts
import { useEffect, useState } from 'react';

type Theme = 'light' | 'dark' | 'system';

export function useTheme() {
  const [theme, setTheme] = useState<Theme>(() => {
    return (localStorage.getItem('bizroom-theme') as Theme) || 'system';
  });

  useEffect(() => {
    const root = document.documentElement;
    const systemDark = window.matchMedia('(prefers-color-scheme: dark)');

    function applyTheme(t: Theme) {
      if (t === 'dark' || (t === 'system' && systemDark.matches)) {
        root.classList.add('dark');
      } else {
        root.classList.remove('dark');
      }
    }

    applyTheme(theme);
    localStorage.setItem('bizroom-theme', theme);

    const listener = () => { if (theme === 'system') applyTheme('system'); };
    systemDark.addEventListener('change', listener);
    return () => systemDark.removeEventListener('change', listener);
  }, [theme]);

  return { theme, setTheme };
}
```

```typescript
  clo: { light: '#767676', dark: '#9E9E9E', name: 'Bradley', icon: '⚖️', role: 'CLO' },
} as const;

export type AgentKey = keyof typeof AGENT_COLORS;

export function getAgentColor(agent: AgentKey, isDark: boolean): string {
  return isDark ? AGENT_COLORS[agent].dark : AGENT_COLORS[agent].light;
}

export function getAgentTint(agent: AgentKey, isDark: boolean): string {
  const color = getAgentColor(agent, isDark);
  const alpha = isDark ? '15' : '08';
  return `${color}${alpha}`;
```
