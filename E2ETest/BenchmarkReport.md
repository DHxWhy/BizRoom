## GPT Realtime 1.5 — 실측 결과

| 항목             | 측정값                                                                |
| ---------------- | --------------------------------------------------------------------- |
| **모델**         | `gpt-4o-realtime-preview`                                             |
| **프로토콜**     | WebSocket (`wss://api.openai.com/v1/realtime`)                        |
| **TTFB**         | **980ms** (첫 텍스트 delta 도착)                                      |
| **전체 완료**    | **1,827ms**                                                           |
| **Delta 이벤트** | 104개                                                                 |
| **응답 품질**    | "디지털 채널 60% 투자, 오프라인 30%, 데이터 분석 10%" — 구체적, 자연스러운 한국어 |
| **특이사항**     | temperature 최소 0.6 (0.5 이하 불가), modalities: ["text"] 사용       |

### Realtime이 빠른 이유
- **WebSocket 연결 유지**: HTTP 요청/응답 오버헤드 없음
- **양방향 스트리밍**: 서버가 준비되는 즉시 delta 전송 시작
- **음성 최적화 모델**: 저지연 추론 아키텍처

### 현재: 순차 호출

```
Hudson (COO) ────[4.8초]────▶

### 개선안: 병렬 호출

```

| 방식   | Sonnet 4.6 (3명) | Haiku 4.5 (3명) | GPT-4o (3명)   | Realtime (1명) |
| ------ | ---------------- | --------------- | -------------- | -------------- |

| 병렬   | **~5초**         | **~2.5초**      | **~2.5초**     | **~1.8초**     |

> 병렬 호출 시 가장 느린 에이전트 기준으로 전체 시간 결정. SSE 스트리밍은 에이전트별 인터리빙 가능.

## 응답 샘플 (동일 프롬프트)

### Realtime (gpt-4o-realtime-preview) — 1.8초
> Q2 마케팅 예산 3천만 원이라면, 디지털 채널에 60%를 집중 투자해 온라인 광고와 SNS 캠페인을 강화하는 게 좋습니다. 나머지 30%는 오프라인 이벤트나 프로모션에 할당해 고객 체험을 유도하고 브랜드 인지도를 높일 수 있습니다. 마지막 10%는 데이터 분석과 A/B 테스트에 사용해 전략의 효율성을 지속적으로 개선하는 데 활용하는 걸 추천합니다.

### Sonnet 4.6 — 4.8초
> Q2 마케팅 예산 3,000만 원 채널 배분 전략: 퍼포먼스 마케팅(Meta/Google) 40% (1,200만 원), 콘텐츠/인플루언서 35% (1,050만 원), SEO/이메일/리타겟팅 25% (750만 원)

### Haiku 4.5 — 2.3초
> 디지털 채널(소셜, 검색, 디스플레이) 50%(1,500만 원), 전통 매체(TV, 라디오) 25%(750만 원), 브랜드 활동 및 기타 채널에 25%(750만 원) 할당하여 효과적으로 예산을 확보하겠습니다.

### GPT-4o — 2.1초
> 온라인 채널에 60%, 오프라인 채널에 40%를 배분합니다. 온라인에서는 소셜 미디어와 검색 광고를 병행하고, 오프라인에서는 이벤트와 협업 사례를 활용합니다.

## 권장 설정

### 해커톤 데모용 (속도+인상 극대화)

```bash
# 음성: Realtime (TTFB <1초)
# 텍스트: GPT-4o (2초) 또는 Haiku (2.3초)
# 시각화: Haiku (빠른 파싱)
ANTHROPIC_MODEL_PREMIUM  = claude-haiku-4-5-20251001
ANTHROPIC_MODEL_BALANCED = claude-haiku-4-5-20251001
ANTHROPIC_MODEL_FAST     = claude-haiku-4-5-20251001
```

예상: 에이전트당 ~2초, 3명 순차 ~7초, 병렬 시 ~2.5초

### 프로덕션 (품질·속도 균형)

```bash
ANTHROPIC_MODEL_PREMIUM  = claude-sonnet-4-6
ANTHROPIC_MODEL_BALANCED = claude-haiku-4-5-20251001
ANTHROPIC_MODEL_FAST     = claude-haiku-4-5-20251001
```

예상: 에이전트당 ~3-5초, 3명 순차 ~12초, 병렬 시 ~5초

### 음성 회의 최적

```bash
# Live 모드 PTT → GPT Realtime (WebSocket, TTFB <1초)
# 텍스트 채팅 → Sonnet 4.6 (~5초)
# Sophia 시각화 → Haiku 4.5 (~2초)
```
